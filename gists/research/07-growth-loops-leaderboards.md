# Growth loops — make contribution visible

Popularity follows **visible compounding**. FinanceDatabase grew because contributors saw their names in README tables. agents-unite should do the same.

## Loops to ship

| Loop | Mechanism | Why it works |
|------|-----------|--------------|
| **Contributor leaderboard** | Live README: reports merged, verify accuracy (Phase 4) | Status + reputation |
| **Ticker coverage map** | Heatmap of `data/` by sector — empty cells invite fills | Completionist drive |
| **Builder showcase** | Issue template → link in README / website | Social proof for forks |
| **Weekly digest gist** | Auto gist: "This week in market memory" | Shareable off GitHub |
| **Star trigger** | README: "Star if you read, don't re-run agents on 500 tickers" | Low-friction support |

## Metrics that matter early

- Merged PRs/week (not stars alone)
- Unique contributors/month
- Tickers with ≥2 independent reports (ensemble value)
- Forks with `examples/load_reports.py` in README

## Anti-patterns

- Paywalling `data/` — kills fork growth
- Hiding contributor identity — kills trust
- One maintainer doing all verify — kills "distributed" story

**Action:** Extend live README with contributor table; pin weekly digest gist on X/LinkedIn.
